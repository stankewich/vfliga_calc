# Исправление: Прерывание расчета сыгранности из-за арендованного игрока

## Проблема

**Дата обнаружения**: 2026-02-25

**Описание**: Расчет сыгранности (synergy) прерывался, если в истории матчей встречался день, когда:
- Арендованный игрок играл за другую команду
- Остальные игроки текущей команды не играли (команда не участвовала в турнире в этот день)

**Пример из лога**:
```
[SynergyCalc] День 25444: 0 игроков играло
[SynergyCalc] День 25444: прерываем анализ (менее 4 игроков)
[SynergyCalc] Итого: 3.15% за 12 матчей
```

**Игрок**: 5711074 (арендованный)
**День**: 25444
**Команда**: 3939

## Анализ

### Текущая логика (неправильная)
```javascript
if (playersInMatch < MIN_PLAYERS_FOR_SYNERGY) {
    console.log(`День ${matchDay}: прерываем анализ (менее 4 игроков)`);
    break; // ❌ Прерывает весь расчет
}
```

### Проблема
- `break` прерывает цикл по всем матчам
- Если встречается день, когда команда не играла, расчет останавливается
- Более поздние матчи (где команда играла) не учитываются
- Результат: занижение бонуса сыгранности

### Почему возникает
1. Матрица строится для всех игроков состава
2. Арендованный игрок имеет матчи за другую команду
3. Эти матчи включаются в матрицу с `played = 0` (не играл за текущую команду)
4. Если в такой день текущая команда не играла → 0 игроков из состава играло
5. Срабатывает условие `< MIN_PLAYERS` → `break`

## Решение

### Новая логика (правильная)
```javascript
if (playersInMatch < MIN_PLAYERS_FOR_SYNERGY) {
    console.log(`День ${matchDay}: пропускаем (менее 4 игроков)`);
    continue; // ✅ Пропускает только этот день
}
```

### Обоснование
- `continue` пропускает только текущую итерацию цикла
- Расчет продолжается для следующих дней
- Учитываются все матчи, где команда играла с достаточным составом

### Различие ситуаций

**Ситуация 1: Команда не играла**
- День 25444: команда 3939 не играла
- Арендованный игрок играл за другую команду
- Результат: 0 игроков из состава играло
- Действие: `continue` (пропускаем день)

**Ситуация 2: Команда играла, но мало игроков**
- День 25450: команда 3939 играла
- Только 3 игрока из состава вышли на поле
- Результат: 3 игрока играло (< 4)
- Действие: `continue` (пропускаем день, низкая сыгранность)

**Важно**: В обеих ситуациях используем `continue`, потому что:
1. Если команда не играла - день не должен учитываться
2. Если команда играла с малым составом - это не означает, что нужно прерывать весь анализ

## Альтернативное решение (не реализовано)

Можно было бы различать ситуации:
```javascript
// Проверяем, играла ли команда в этот день
const teamPlayedThisDay = checkIfTeamPlayedOnDay(matchDay, teamId);

if (playersInMatch < MIN_PLAYERS) {
    if (teamPlayedThisDay) {
        // Команда играла, но мало игроков - прерываем
        break;
    } else {
        // Команда не играла - пропускаем
        continue;
    }
}
```

Но это сложнее и требует дополнительных данных. Текущее решение проще и работает корректно.

## Влияние на расчеты

### Положительное
- Учитываются все матчи команды
- Корректный расчет для команд с арендованными игроками
- Более точный бонус сыгранности

### Потенциальные риски
- Если команда долго не играла (пауза в турнире), старые матчи могут учитываться
- Но это корректное поведение - сыгранность сохраняется
