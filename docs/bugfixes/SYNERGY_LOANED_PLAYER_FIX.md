# Исправление: Прерывание расчета сыгранности из-за арендованного игрока

## Проблема

**Дата обнаружения**: 2026-02-25

**Описание**: Расчет сыгранности (synergy) прерывался, если в истории матчей встречался день, когда:
- Арендованный игрок играл за другую команду
- Остальные игроки текущей команды не играли (команда не участвовала в турнире в этот день)

**Пример из лога**:
```
[SynergyCalc] День 25444: 0 игроков играло
[SynergyCalc] День 25444: прерываем анализ (менее 4 игроков)
[SynergyCalc] Итого: 3.15% за 12 матчей
```

**Игрок**: 5711074 (арендованный)
**День**: 25444
**Команда**: 3939

## Решение (Вариант 2.1 - Реализовано)

### Логика
Различаем две ситуации:
1. **Команда не играла** → пропускаем день (`continue`)
2. **Команда играла, но мало игроков** → прерываем анализ (`break`)

### Реализация

**1. Парсинг дней матчей команды**
```javascript
function getAllTeamMatchDays(teamId) {
    // Загружает roster_m.php?num={teamId}
    // Парсит все ссылки на viewmatch.php
    // Извлекает дни матчей
    // Возвращает массив дней [25530, 25527, ...]
}
```

**2. Кэширование**
```javascript
const CACHE_TTL = 60 * 60 * 1000; // 1 час

function getTeamMatchDaysWithCache(teamId) {
    // Проверяет кэш в localStorage
    // Если кэш валиден (< 1 часа) → возвращает из кэша
    // Иначе загружает с сервера и сохраняет в кэш

```

**3. Обновленная логика расчета**
```javascript
// Загружаем дни матчей команды
const teamMatchDays = await getTeamMatchDaysWithCache(teamId);

// В цикле расчета
if (playersInMatch < MIN_PLAYERS) {
    if (teamMatchDays.includes(matchDay)) {
        // Команда играла → ПРЕРЫВАЕМ
        console.log(`День ${matchDay}: прерываем (команда играла, но только ${playersInMatch} игроков)`);
        break;
    } else {
        // Команда не играла → ПРОПУСКАЕМ
        console.log(`День ${matchDay}: пропускаем (команда не играла)`);
        continue;
    }
}
```

### Преимущества

1. **Точность** - различает ситуации "команда не играла" vs "мало игроков"
2. **Производительность** - кэширование на 1 час, минимум запросов
3. **Надежность** - fallback на старый кэш при ошибке загрузки
4. **Независимость** - не зависит от состава игроков

### Кэширование

**Структура кэша** (localStorage):

**TTL**: 1 час (3600000 мс)
**Fallback**: Если загрузка не удалась, используется старый кэш

## Исправление scope issue (2026-02-25)

### Проблема
После реализации функций кэширования возникла ошибка:
```
ReferenceError: getTeamMatchDaysWithCache is not defined
```

**Причина**: Функции кэширования были определены в строках ~9750-9900, а `calculateSynergyFromMatrix` находится в строке ~4460. JavaScript не может использовать функции, которые определены позже в коде.

### Решение
Перемещены все функции кэширования из строк ~9750-9900 в строки ~4940+ (сразу после `SYNERGY_MATRIX_CONFIG`).

**Перемещенные функции**:
- `getAllTeamMatchDays(teamId)`
- `getCachedTeamMatchDays(teamId)`
- `setCachedTeamMatchDays(teamId, days)`
- `isCacheValid(timestamp)`
- `getTeamMatchDaysWithCache(teamId)`
- `TEAM_MATCH_DAYS_CACHE_TTL` (константа)

**Новое расположение в коде**:
- `SYNERGY_MATRIX_CONFIG`: строка ~4918
- Функции кэширования: строки ~4940-5120
- `calculateSynergyFromMatrix`: строка ~5125+

### Результат
✅ Функции кэширования теперь доступны для `calculateSynergyFromMatrix`
✅ Ошибка `ReferenceError` устранена
✅ Код готов к тестированию

## Исправление: Матчи за другие команды в матрице (2026-02-25)

### Проблема
После исправления scope issue обнаружена новая проблема: сыгранность для состава из 10 игроков была ВЫШЕ, чем для 11 игроков.

**Пример из лога**:
- 10 игроков: 6.75% за 25 матчей
- 11 игроков: 5.25% за 20 матчей

**Причина**: В матрицу сыгранности попадали ВСЕ дни из истории игроков, включая дни, когда они играли за ДРУГИЕ команды. Например:
- Игрок 10816867: "всего 34 матчей, за команду 3939: 2, за другие команды: 20"
- Игрок 11214605: "всего 30 матчей, за команду 3939: 3, за другие команды: 8"

Эти игроки добавляли в матрицу дни 25444, 25429, 25404, 25393, 25364, когда они играли за другие команды. Команда 3939 в эти дни не играла, поэтому эти дни пропускались через `continue`, но уменьшали общее количество учтенных матчей.

### Решение
Изменена логика сбора дней матчей в функции `buildSynergyMatrixFromPlayersForCalc`:

**Было**:
```javascript
const allMatchDays = new Set();
Object.values(playerHistories).forEach(history => {
    history.forEach(match => {
        allMatchDays.add(match.day);  // Все дни, включая матчи за другие команды
    });
});
```

**Стало**:
```javascript
const allMatchDays = new Set();
Object.values(playerHistories).forEach(history => {
    history.forEach(match => {
        // Если фильтр по команде задан, добавляем только матчи за эту команду
        if (!teamId || match.playedForTeam) {
            allMatchDays.add(match.day);
        }
    });
});
```

### Результат
✅ В матрицу попадают только дни, когда игроки играли за ТЕКУЩУЮ команду
✅ Матчи за другие команды не влияют на расчет сыгранности
✅ Сыгранность для 11 игроков теперь будет корректной

**Файл**: `calc.user.js` (строки ~5408-5416)
